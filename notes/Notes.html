Basic git commands

git init <name of project>

git status → The git status command shows the state of the working directory and the staging area. It allows you to see staged changes and the files that aren’t being tracked by Git. The Status output does not display any information about the committed project history.

git log → List commits that are reachable by following the parent links from the given commit(s), but exclude commits that are reachable from the one(s) given with a ^ in front of them. The output is given in reverse chronological order by default.

  • <to>....<from>

  • — <file> -log of changes to filer

git-ls-files - Show information about files in the index and the working tree

git push <remote> <branch>

GIT ADD

git add → add files to staging area for commit

  • -A → Update the index not only where the working tree has a file matching <pathspec> but also where the index already has an entry. This adds, modifies, and removes index entries to match the working tree.

Git renaming and removing

  • git mv <source> <target>
    • To undo: just rename back to original
  • git rm <file> (works on files tracked by git)
    • if we use a bash command, the change is staged in the staging area

Committing

git commit

  • Commands
    • -m - write message directly in console
    • -am add and commit at same time
    • a, --all commit all changed files -i, --include add specified files to index for commit --interactive interactively add files -p, --patch interactively add changes -o, --only commit only specified files -n, --no-verify bypass pre-commit and commit-msg hooks --dry-run show what would be committed --short show status concisely --branch show branch information --ahead-behind compute full ahead/behind values --porcelain machine-readable output --long show status in long format (default) -z, --null terminate entries with NUL --amend amend previous commit

Deleting a Commit

  • git reset --soft HEAD~1 →Delete the most recent commit, keeping the work you've done:
  • git rest —hard HEAD~1 →Delete the most recent commit, destroying the work you've done:
  • git checkout — <file-name> reverts file back to original state, removes it from commit area

    Aborting a rebase

    git rebase —abort

    Continue rebase after resolving conflict:

    git rebase —continue

    Combine commits with rebase

    • git rebase -i HEAD~#
    • git push --force origin HEAD to push changes after rebase
    • git push --force-with-lease origin HEAD
      • considered a safer option that will not overwrite the work done on the remote branch in case more commits were attached to it (for instance, by another developer). Moreover, it helps you to avoid overwriting another developer's work by force pushing.

    Git Pull - With Rebase (and git fetch)

    • git fetch - a non-destructive command that updates the references between the remote repository and the local repository. In other words, pulls down refs and checks to see if there is any divergence. 
    git fetch //fetches the refs in remote
//pulls down changes from remote while keeping your 
//changes at the front
git pull rebase --origin main

    Git cheat sheet:

    • Git init creates a git repository in your present working directory
    • Git clone <URL> fetch remote repository and data (use only once at the start)
    • Git pull <remote> <branch> fetch and merge remote changes
    • Git add . adds all changes to be committed
    • Git commit -m “<message>” commits your staged changes locally
    • Git remote -v displays remote directories
    • Git remote rm <name>  removes the remote
    • Git remote add <name> <URL> adds a new remote
    • Git reset --hard
      • Reset current directory to HEAD
    • Git push <remote> <branch> push latest commit
    • Git --set-upstream <remote> <branch> sets your default branch
      • Now just say “Git push” and it will know which remote and branch
    • Git branch Tells you which branch you’re on
      • Follow with r to display remote branches
      • Follow with -merged or -no-merged to sort by merged branches
    • Git branch <name> Create new branch off of current branch
    • Git checkout <branch> Switch branches (will change your current project in Unity)
      • Git checkout <branch> -- <Path/DirectoryOrFile> checkout only a file
    • Git merge <branch> Bring changes from target branch to current branch and merge
    • Git rebase <branch> checkout and merge
    • Git log display git commit history for current branch
      • Git log <remote>/<branch> log remote changes
    • Git show origin/<branch> see exact changes on remote
    • Git show REVISION:src/main.ext see previous version of a file
      • Example: git show HEAD~2:assets/_scripts/example.cs
      • This shows the exact source code for that file via the terminal
      • Use the f and b key to navigate the file reader
    • Handle Conflicts
      • Git checkout --theirs / --ours (*.filename or files)
      • Git add . (add changes from your current directory)
      • Git merge --continue

      To set a new url if repo has been moved or renamed

      git remote set-url origin <url>
